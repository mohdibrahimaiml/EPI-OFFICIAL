The EPI Thesis

The Evidence Container

One file. Complete proof.

Every consequential AI decision sealed into a single, portable, cryptographically signed artifact.

🔒

Cryptographic Signing

Ed25519 signed manifest. SHA-256 file integrity. Every byte change invalidates the signature. Tamper-evident by construction.

manifest.json ← Ed25519 signature
🔗

Hash-Chained Timeline

Every step in steps.jsonl is linked to its predecessor via prev_hash. Insert, delete, or reorder — the chain breaks. Immutable by design.

steps.jsonl ← prev_hash chain
📦

Polyglot Container

A .epi file is valid HTML AND a binary evidence archive. Double-click opens in any browser. Zero install. Zero upload. Fully offline.

.epi ← HTML + ZIP + signature
Verification

Five independent checks

No server. No trust. No installation. Just a browser and a .epi file.

Structure

Valid envelope, extractable ZIP, required files present.

Integrity

Every file hash in file_manifest matches computed SHA-256.

Signature

Ed25519 signature valid against canonical manifest hash.

Chain

prev_hash links intact. Monotonic indices and timestamps.

SCITT

COSE_Sign1 statement + receipt verified against ledger.

Integration

Three lines of Python

No monkey patching. Streaming and non-streaming both supported. Works with any LLM SDK.

agent.py
from openai import OpenAI
from epi_recorder import record, wrap_openai

client = wrap_openai(OpenAI())

with record("agent.epi", goal="Assess loan application"):
    response = client.chat.completions.create(
        model="gpt-4",
        messages=[{"role":"user", "content":"Assess this application"}]
    )

# Every call → signed .epi evidence
# epi verify agent.epi → VERIFIED ✓
Compliance

Built for the EU AI Act

Article 12 record-keeping. Article 14 human oversight. Article 19 transparency. All addressable from a single .epi artifact.

Article 12

Record-Keeping

Seals the complete execution trail — every LLM call, tool invocation, decision, and environment snapshot — into one signed artifact. Auditors open it in a browser.

Article 14

Human Oversight

review.json is appended post-sealing without breaking the cryptographic signature. Who reviewed, when, what they decided — in the same signed file.

Article 19

Transparency

policy.json and policy_evaluation.json travel with the container. Every rule applied, every threshold checked, every outcome logged — inspectable and auditable.
