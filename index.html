Inside a .epi File: Complete Anatomy

Every .epi file is a self-contained ZIP archive with everything needed to prove, reproduce, and audit an AI workflow.

üì¶ .epi (application/epi+zip) ‚ñº
üìÑ mimetype ‚ñº
Content: "application/epi+zip" (stored, uncompressed)
Purpose: Identifies file as EPI container (like PDF magic bytes)
üìã manifest.json ‚ñº
Contains:
  • Workflow metadata (id, created_at, author)
  • Environment info (OS, Python version, dependencies)
  • Entrypoint (command or script executed)
  • Replay policy (offline_only/prefer_offline/allow_network)
  • Digital signature (Ed25519)
  • Hash verification info
Description: Global header of the package; like PDF catalog
üìù steps.jsonl ‚ñº
Format: NDJSON (newline-delimited JSON)
Contains:
  • Chronological timeline of all steps
  • Shell commands (with stdout/stderr/exit code)
  • Python function calls (params, return values)
  • LLM calls (model, prompt, response, latency)
  • HTTP requests (URL, method, headers, response)
  • File I/O operations (read/write paths, hashes)
  • Each step immutably hashed (SHA-256)
Description: Immutable execution timeline; the heart of reproducibility
üìÅ artifacts/ ‚ñº
Structure: sha256_<hash>.<extension>
Contains:
  • All outputs captured during execution
  • Files, logs, plots, images, models
  • Content-addressed (hash-based naming)
  • Tamper-evident structure
Description: All workflow outputs preserved
üíæ cache/ ‚ñº
Structure: <provider>/<key>.json
Contains:
  • Cached API responses (OpenAI, Anthropic, etc.)
  • Full LLM request/response pairs
  • Enables deterministic replay
Example: cache/openai/chat_completion_abc123.json
Description: Why EPI enables $0-cost reproduction
üåç env.json ‚ñº
Contains:
  • Operating system (OS, version)
  • Python version and packages
  • GPU/hardware info
  • Environment variables (redacted)
  • Docker/container info
Description: Environment snapshot for reproducibility
‚úì checks.json ‚ñº
Verification DSL rules:
  • "assert file exists: path/to/file"
  • "response contains: search_string"
  • "exit code: 0"
  • "LLM response matches: expected_output"
  • Custom validation logic
Description: Automated verification tests
üëÅÔ∏è viewer/ ‚ñº
Contains:
  • Embedded static HTML + JS
  • index.html (entry point)
  • CSS/JS for timeline visualization
  • No code execution (read-only)
  • Works offline in any browser
Description: Safe, portable evidence viewer
üîê signatures/ ‚ñº
Contains:
  • Detached signatures
  • Ed25519 public key
  • Cryptographic attestations
  • Signer identity information
Description: Legal-grade proof of authenticity
üîó provenance.json ‚ñº
Contains:
  • Git repository refs/commits
  • Dataset IDs and versions
  • Model checksums
  • Data lineage
  • Author attribution
Description: Full chain of custody
üîí attachments/ ‚ñº
Contains:
  • Encrypted large blobs (optional)
  • AES-GCM encryption
  • Compression support
Description: For confidential or large files

No external dependencies. No proprietary formats. Pure evidence.
