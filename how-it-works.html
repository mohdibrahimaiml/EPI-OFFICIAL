The EPI Lifecycle

Transform an AI run into a self-contained evidence artifact that captures execution, preserves trust, and can now include policy, fault analysis, and human review.

📥

0. Install

Get started in seconds. Install the epi-recorder package from PyPI with a single command.

$ pip install epi-recorder

✨ New in v2.8.4: policy-aware artifacts, embedded fault analysis, human review, clearer signed/unsigned/tampered trust states, and safer default CLI behavior. See Get Started for details.

📹

1. Record ✅ Shipped

Use explicit recording APIs or supported wrappers to capture execution into a single sealed file.

$ epi record script.py
📏

2. Apply Policy

Teams can define rules once and embed them with the run, so reviewers later see the exact policy used at execution time.

$ epi policy init --profile finance.loan-underwriting
🧠

3. Analyze Faults

EPI can write analysis.json before sealing so the artifact contains the primary fault, violated rule, and why the outcome needs review.

$ epi analyze scan_01.epi
👤

4. Review

A human reviewer can confirm or dismiss the finding and append that judgment as review.json without breaking the evidence chain.

$ epi review scan_01.epi
🛡️

5. Verify & Share ✅ Shipped

Open the artifact and see trust state, timeline, policy, analysis, and review together. Signed, unsigned, and tampered states are explicit.

$ epi view output.epi

Under the Hood

Enterprise-grade architecture. No proprietary lock-in.

📦 ZIP-Based Container

Self-contained .epi files. Store evidence + viewer in one portable archive.

🔐 Ed25519 Signatures

Same crypto as Signal & SSH. Tamper-proof, military-grade security.

🗂️ Content-Addressed Storage

SHA-256 hashing ensures file integrity. Deduplication by design.

🦙 Multi-Provider Support

OpenAI, Anthropic, Ollama, Google Gemini. Automatic capture.

🌐 Offline Replay

Cached LLM responses enable deterministic re-runs without API calls.

📊 Embedded Viewer

Static HTML/JS timeline. No server required. Works like a PDF.
